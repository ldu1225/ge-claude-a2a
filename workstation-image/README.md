# 워크스테이션 컨테이너 이미지 (Workstation Custom OS Image)

본 디렉토리는 **사용자별 독립된 가상 머신(Cloud Workstation)**의 뼈대가 되는 **커스텀 도커(Docker) 컨테이너 이미지**의 구성 명세 및 빌드 패키지입니다.

이 이미지는 구글 클라우드가 제공하는 공식 웹 IDE(Code OSS) 환경 위에 **자율 코딩 에이전트(Claude Code CLI, Gemini CLI)**와 이들을 조율할 **로컬 A2A 서버 데몬**, 그리고 **최적의 코딩 가이드 스킬 파일**을 정교하게 사전 베이킹(Pre-baking)하여 단일 패키지 형태로 제작됩니다.

---

## 📂 1. 베이킹 도구 및 아키텍처 스택

커스텀 이미지는 대기업 보안 가이드라인에 맞추어 완전 통제된 프라이빗 환경으로 컴파일됩니다:

* **베이스 이미지:** `us-central1-docker.pkg.dev/cloud-workstations-images/predefined/code-oss:latest`
  * 구글 클라우드가 직접 유지 관리하고 서명한 공식 웹 VS Code 환경입니다.
* **설치되는 개발 도구 명세:**
  1. **Node.js 24+ & npm:** 고성능 자율 실행 엔진 구동용 런타임.
  2. **Claude Code CLI (`@anthropic-ai/claude-code`):** Anthropic Vertex AI 모델을 호출하여 스스로 셸 명령어를 실행하고 코드를 편집하는 자율 코딩 에이전트.
  3. **Gemini CLI (`@google/gemini-cli`):** 구글의 멀티모달 모델을 터미널에서 신속하게 제어하기 위한 공식 CLI.
  4. **GitHub CLI (`gh`):** 코딩 완료 후 PR 생성 및 이슈 관리를 자동화하기 위한 깃허브 공식 도구.
  5. **A2A 서버 데몬:** 라우터와 동일한 엔진으로, 로컬 모드(`local`)로 기동하여 대화 세션을 디스크에 캐싱하고 에이전트 서브프로세스의 입출력 스트림 파이프를 실시간 라우팅합니다.

## 🎯 2. 사전 제공 스킬 파일 구성 (Home Skeleton)

사용자 가상 머신이 최초 생성될 때, 사용자의 홈 디렉토리(`/home/user`)에 **AI 에이전트 전용 코딩 지침서**가 자동으로 주입됩니다:
* **주입 경로:** `/home/user/.claude/skills/a2ui-rich-ui/SKILL.md` (Docker 이미지의 `skel/` 디렉토리를 통해 복제)
* **스킬의 핵심 역할:**
  * Claude Code가 사용자의 코딩 요청을 받았을 때, 임의로 행동하지 않고 **우리가 사전에 설계한 극도로 화려하고 정교한 맞춤형 A2UI 컴포넌트 규격대로 대시보드 카드를 설계하도록 통제**합니다.
  * 불필요한 파일 목록이나 긴 코드 원본을 메신저 창에 중복 노출하는 행동을 원천 금지시켜 쾌적하고 압도적인 가독성을 상시 보장합니다.

## 🔄 3. 자동 부팅 데몬 제어 스크립트 (`300_start-a2a.sh`)

가상 머신 인스턴스가 켜질 때, 컨테이너 부팅 단계(`/etc/workstation-startup.d/`)에서 **자동 기동 스크립트**가 백그라운드로 자동 가동됩니다:

### 스크립트 핵심 가동 파이프라인:
1. **사용자 권한 보장 (Security Rule):** 스크립트 자체는 root 권한으로 트리거되나, 내부 A2A 서버 프로세스는 즉시 일반 사용자(`user`) 권한으로 전환(`runuser -u user`)하여 구동됩니다. 이를 통해 에이전트가 자동 코딩한 파일의 소유권이 root로 꼬여 사용자가 IDE에서 파일을 편집하지 못하는 현상을 완전히 예방합니다.
2. **스킬 실시간 동기화:** 부팅될 때마다 이미지 내의 마스터 스킬 폴더에서 사용자의 홈 폴더(`.claude/skills/`)로 최신 A2UI 가이드 파일을 실시간 덮어쓰기하여, 언제나 최신 패치 지침이 동작하도록 동기화합니다.
3. **영속성 자동 복구 슈퍼바이저 (Supervisor Loop):** 
   * A2A Node 서버가 예기치 않게 비정상 종료되거나 에러를 뱉을 경우, 백그라운드 슈퍼바이저 루프(PID 관리자)가 이를 **실시간으로 감지하여 단 1초 만에 즉각 자동 복구 및 재시작(Respawn)**시킵니다.
4. **유휴 자동 꺼짐 모니터 (Idle Watchdog):** 
   * 포트 8080에 15분(900초) 동안 어떠한 API 요청도 유입되지 않으면, 대기 중인 인스턴스를 자동으로 종료(Stop)시켜 클라우드 과금을 안전하게 방어합니다.

## 🚀 4. 클라우드 빌드 및 이미지 배포

구글 클라우드 빌드(Cloud Build) 엔진을 사용하여, 소스코드를 안전하게 격리된 샌드박스에서 빌드한 뒤 비공개 Artifact Registry 저장소로 푸시합니다.

```bash
# 1. workstation-image 디렉토리로 이동
cd workstation-image

# 2. GCP 프로젝트 ID를 지정하여 빌드 실행 (build.sh 호출)
PROJECT_ID=your-gcp-project-id ./build.sh
```

## 🐛 5. 가상 머신 내부 디버깅 및 실시간 로그 분석

데모 도중 에이전트의 작동 상태나 빌드 런타임 현황을 직접 정밀 모니터링해야 할 경우, 다음의 디버깅 커맨드셋을 활용합니다:

### 1. 실시간 A2A 통신 로그 스트리밍 판독
가상 머신 내의 A2A 서버 로그는 다음 경로에 실시간 누적됩니다:
```bash
# 워크스테이션 터미널 접속 후 실시간 로그 스트리밍 확인
tail -f /var/log/a2a-server.log
```

### 2. 가상 머신 컨테이너 내부 프로세스 트리 확인
에이전트 데몬과 Claude CLI 서브프로세스가 안전하게 활성화되어 있는지 확인합니다:
```bash
ps -ef | grep node
# 또는 아래 명령어로 A2A 부팅 유닛의 종합 상태 확인
journalctl -u workstation-startup -n 100 -f
```

### 3. 강제 핫-리로드 및 데몬 수동 재시작
라우터 코드가 업데이트되었을 때 가상 머신을 껐다 켜지 않고 컨테이너 내에서 데몬만 수동으로 리셋하는 법:
```bash
# 1. 실행 중인 Node 서버를 강제 종료 (슈퍼바이저 데몬이 1초 만에 최신 코드로 즉시 재시작시킴)
pkill -9 -f "node /opt/a2a-server/dist/index.js"

# 2. 만약 유휴 차단 타이머로 인해 슈퍼바이저까지 퇴근한 경우, 부팅 유닛 자체를 강제 재실행
sudo /etc/workstation-startup.d/300_start-a2a.sh
```
